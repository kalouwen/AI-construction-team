# AI Construction Team 🏗️

> 一句话：**给任何代码工程装修成 AI 能高效干活的精装房。**

你是老板，你的客户是各种"毛坯房"（代码工程）。AI Construction Team 是你的装修公司——全套工具、标准材料、训练有素的工人、完整的施工流程。

## 它解决什么问题

AI 进一个陌生项目，通常会遇到：
- **读不懂** — 没文档，不知道代码结构
- **不敢动** — 不知道构建命令，改了不知道会不会坏
- **没法验** — 没测试，改完只能祈祷
- **回不去** — 没有 git 规范，出了问题没法回滚
- **没记忆** — 每次对话从零开始，经验不积累

AI Construction Team 通过**环境层 → 迭代层 → 自治层**三层架构，系统性解决这些问题。

## 施工流程

```
老板说："去给 XX 工程装修一下"
        │
        ▼
   /survey ── 踏勘：扫描工地，生成项目画像，打 AI 友好度分
        │
        ▼
  /mode-plan ── 策划：大目标拆成原子任务，列施工清单
        │
        ▼
 /mode-deploy ── 施工：装 hooks → 配 rules → 建知识库 → 装 skills
        │
        ▼
    /check ── 验收：逐项检查，确认质量达标
        │
        ▼
 /mode-learn ── 复盘：今天踩了什么坑？提炼成新规则，更新材料库
```

| 岗位 | 职责 |

| `session-start` | 值班室 — 每天开工先汇报昨天进度 |
| `pre-compact-inject` | 保存进度 — 上下文压缩前先存好关键状态 |
| ... | 还有 5 个（格式化/提交验证/编辑复查等） |

## 工人班组（5 个模板）

每个工人有**专长、权限范围、协作网络、明确禁区**：

| 工人 | 模型 | 专长 | 禁区 |
|------|------|------|------|
| **planner** | opus | 画图纸、出方案 | 绝不亲自改代码 |
| **research** | sonnet | 到处看、到处查 | 绝不替 planner 做决策 |
| **code-reviewer** | sonnet | 逐行检查质量 | 绝不自己修代码 |
| **security-reviewer** | sonnet | 专查安全隐患 | 绝不降级安全等级 |
| **test-writer** | sonnet | 补写测试 | 绝不改源代码 |

## 三层架构

```
┌─────────────────────────────────────┐
│          自治层                       │
│   端到端自动 · 多Agent协同 · 按需找人   │
├─────────────────────────────────────┤
│          迭代层                       │
│   经验沉淀 · 知识保鲜 · 工作方式改进    │
├─────────────────────────────────────┤
│          环境层                       │
│   能读懂 · 能动手 · 能验证 · 能回滚     │
└─────────────────────────────────────┘
```

- **环境层**：AI 能不能干活 — hooks/rules/知识库让 AI 有能力干活
- **迭代层**：AI 能不能越用越好 — evolution-score/instinct-extract 让经验自动积累
- **自治层**：AI 能不能自主完成 — reward-loop/自动化 skills 让 AI 端到端闭环

## 快速开始

1. 克隆到本地
2. 在目标工程旁边打开 Claude Code
3. 运行 `/survey <目标工程路径>` — 踏勘
4. 运行 `/mode-plan` — 出方案
5. 运行 `/mode-deploy` — 施工

## 设计原则

- **所有规则必须有代码执行点** — 不接受纯文字软约束，hook/CI/脚本强制执行
- **通用和专属分开** — `templates/` 是通用材料，目标工程 `.claude/knowledge/` 是专属知识
- **数据驱动进化** — evolution-score 采集信号，evolution-review 分析改进，不靠感觉
- **实事求是** — 不知道就说不知道，方案不行就说不行，不美化不糊弄
