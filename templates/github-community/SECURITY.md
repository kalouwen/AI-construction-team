# Security Policy

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security issues via one of these channels:

1. **GitHub Private Vulnerability Reporting** (preferred)
   Go to the Security tab → "Report a vulnerability"

2. **Email**
   Send details to: [security contact — fill in before deploying]

Include in your report:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## What to Expect

- Acknowledgment within 48 hours
- Status update within 7 days
- Credit in the changelog (unless you prefer anonymity)

## Scope

In scope: anything in this repository that could affect users or their data.

Out of scope: third-party dependencies (report those to their maintainers directly).
