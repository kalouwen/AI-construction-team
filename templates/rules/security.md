# 安全编码规范

## 输入验证
- 所有外部输入（用户输入、API 参数、文件读取）必须验证
- 使用白名单验证，不用黑名单
- SQL 用参数化查询，永远不要拼接

## 敏感数据
- 密钥/密码不硬编码，用环境变量或密钥管理服务
- 日志中不打印敏感信息（密码、token、身份证号）
- 错误信息不暴露内部实现细节

## 认证与授权
- 密码必须 hash 存储（bcrypt/argon2）
- API 接口必须有认证检查
- 权限检查在服务端做，不依赖客户端

## 依赖
- 不引入未审查的第三方库
- 定期检查依赖的已知漏洞
